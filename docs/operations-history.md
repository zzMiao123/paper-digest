## Update Rules

- Update the release-history table when a release is published and the matching
  post-release follow-up issue exists.
- Update the maintainer-operations log when a workflow, issue form, policy, or
  repository-admin rule changes how maintainers operate the project.
- Do not silently backfill unknown history. Use `Not tracked` when earlier
  release-cycle artifacts did not exist.
- If a change is significant enough to affect future maintainers, update this
  file in the same pull request that changes the process.
